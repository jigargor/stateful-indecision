# Security Policy

## Reporting a vulnerability

Please report vulnerabilities privately to the maintainers instead of opening a public issue.

Include:
- affected version(s) and commit SHA when possible
- reproduction steps
- expected and observed behavior
- impact assessment

## Scope and response goals

- We prioritize vulnerabilities that can break ledger integrity, path-firewall boundaries, or safety gate behavior.
- Initial triage target: 3 business days.
- Fix timeline depends on severity and exploitability.

## Coordinated disclosure

Please allow maintainers time to investigate, patch, and prepare release notes before public disclosure.
